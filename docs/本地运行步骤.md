# 本地运行步骤

## 1. 安装依赖

Windows PowerShell 下建议使用 `npm.cmd`，避免 `npm.ps1` 被执行策略拦截：

```bash
npm.cmd install
```

## 2. 配置环境变量

复制 `.env.example` 为 `.env.local`，并填入 Supabase 项目信息：

```bash
copy .env.example .env.local
```

需要配置：

- `NEXT_PUBLIC_SUPABASE_URL`：Supabase 项目 URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`：Supabase anon public key
- `SUPABASE_SERVICE_ROLE_KEY`：仅服务端使用，用于关键写入
- `NEXT_PUBLIC_SITE_URL`：本地默认 `http://localhost:3000`
- `DEFAULT_CAMP_SLUG`：V1 默认训练营，填写 `ai-life-os-camp`

不要把真实密钥发送到聊天、提交到代码仓库或写入文档。

## 3. 初始化数据库

在 Supabase SQL Editor 中执行：

```text
supabase/migrations/0001_mvp_v1_schema.sql
```

再按顺序执行：

```text
supabase/migrations/0002_supabase_security.sql
supabase/seed.sql
```

`0001` 是可迁移的标准 PostgreSQL 业务 Schema，不依赖 Supabase
`auth.users`。`0002` 只负责 Supabase 的服务端函数权限。

首次注册主理人账号并完成登录后，如需开通后台权限：

1. 在 `app_users` 中找到该账号的内部用户 ID。
2. 复制 `supabase/bootstrap-owner.sql`。
3. 在副本中将 `<OWNER_INTERNAL_USER_ID>` 替换为内部用户 ID。
4. 只执行一次，不要把真实用户 ID 写回模板文件。

## 4. 启动本地服务

```bash
npm.cmd run dev -- -p 3000
```

访问：

```text
http://localhost:3000
```

## 5. 运行测试

```bash
npm.cmd test
```

## 6. 生产构建

```bash
npm.cmd run build
```

## 7. 开发服务注意事项

不要在 `npm.cmd run dev` 正在运行时同时执行 `npm.cmd run build`。

Next.js 的开发服务和生产构建都会写入 `.next` 目录；如果同时运行，可能导致开发预览出现：

```text
__webpack_modules__[moduleId] is not a function
Cannot find module './xxxx.js'
```

如果本地预览出现类似错误，处理方式：

1. 停止当前 Next.js dev 进程。
2. 删除项目根目录下的 `.next` 目录。
3. 重新运行：

```bash
npm.cmd run dev -- -p 3000
```
