# 阶段 4：签到闭环

## 已实现

- `POST /api/check-ins` 从占位改为服务端写入。
- 使用 Supabase Auth 获取当前用户。
- 使用 service role 服务端执行关键写入，避免前端伪造积分。
- 签到按训练营时区计算自然日。
- 同一学员同一天重复签到不会重复创建签到，也不会重复发积分。
- 签到成功后写入 `point_ledger`，event key 为：

```text
daily_check_in:{enrollmentId}:{localDate}
```

- 签到后根据 V1 徽章规则检查新徽章，并写入 `badge_awards`。
