# 国内生产部署方案

## 1. 部署目标

正式生产环境只部署在中国大陆可稳定访问的云服务器：

- 阿里云轻量应用服务器，或
- 腾讯云轻量应用服务器

运行结构：

```text
域名 + HTTPS
    ↓
Nginx :80/:443
    ↓
PM2
    ↓
Next.js / Node.js :3000
    ↓
国内 PostgreSQL（未来接入）
```

当前 mock H5 MVP 保持不变：

- 不接 Supabase。
- 不新增业务功能。
- 不修改页面逻辑。
- 当前数据继续使用 `mockData.ts + localStorage`。
- PostgreSQL 仅作为未来真实数据阶段的数据库目标。

## 2. 服务器推荐配置

### 推荐配置

| 项目 | 推荐值 |
| --- | --- |
| CPU | 2 核 |
| 内存 | 4 GB |
| 系统盘 | 60 GB SSD |
| 公网带宽 | 5 Mbps 起步 |
| 操作系统 | Ubuntu 22.04 LTS 或 24.04 LTS |
| Node.js | 22 LTS |
| 包管理器 | pnpm 11 |
| 进程管理 | PM2 |
| Web 入口 | Nginx |

2 核 2 GB 可以用于低流量测试，但正式生产建议使用 2 核 4 GB，避免构建 Next.js 时内存不足。

服务器地域应靠近主要用户。应用服务器、未来 PostgreSQL 和对象存储应位于同一地域。

## 3. 需要准备

- 已实名认证的阿里云或腾讯云账号。
- 轻量应用服务器一台。
- 已备案或准备备案的域名。
- 域名 DNS 管理权限。
- GitHub 私有仓库 Deploy Key，或可上传的代码发布包。

中国大陆服务器绑定域名正式提供网站服务前，需要完成 ICP 备案；上线后按要求完成公安备案。

## 4. 首次安装环境

登录服务器：

```bash
ssh root@<SERVER_PUBLIC_IP>
```

更新系统：

```bash
sudo apt update
sudo apt upgrade -y
sudo apt install -y git nginx curl ca-certificates
```

安装 Node.js 22 LTS 后检查：

```bash
node --version
npm --version
```

启用 pnpm：

```bash
sudo corepack enable
corepack prepare pnpm@11.7.0 --activate
pnpm --version
```

安装 PM2：

```bash
sudo npm install -g pm2
pm2 --version
```

## 5. 部署目录和代码

统一使用：

```text
/opt/gamified-bootcamp/current
```

示例：

```bash
sudo mkdir -p /opt/gamified-bootcamp
sudo chown -R $USER:$USER /opt/gamified-bootcamp
git clone <PRIVATE_REPOSITORY_URL> /opt/gamified-bootcamp/current
cd /opt/gamified-bootcamp/current
```

私有仓库凭据、Token、密码和数据库连接串不得写入源码或部署文档。

## 6. Build 命令

生产发布统一执行：

```bash
cd /opt/gamified-bootcamp/current
pnpm install --frozen-lockfile
pnpm test
pnpm build
```

只有测试和生产构建通过后才能重启线上服务。

## 7. Start 命令

不使用 PM2 时，可直接验证生产启动：

```bash
pnpm start --hostname 127.0.0.1 --port 3000
```

检查：

```bash
curl -I http://127.0.0.1:3000/app/home
```

正式环境必须使用 PM2 守护，不使用 `pnpm dev`。

## 8. PM2 配置

项目已准备：

```text
deploy/ecosystem.config.cjs
```

该配置：

- 以生产模式运行 Next.js。
- 监听 `127.0.0.1:3000`。
- 异常退出后自动重启。
- 内存超过 750 MB 时自动重启。
- 日志写入 `/var/log/gamified-bootcamp/`。

准备日志目录：

```bash
sudo mkdir -p /var/log/gamified-bootcamp
sudo chown -R $USER:$USER /var/log/gamified-bootcamp
```

首次启动：

```bash
cd /opt/gamified-bootcamp/current
pm2 start deploy/ecosystem.config.cjs --env production
pm2 status
pm2 logs gamified-bootcamp
```

配置开机启动：

```bash
pm2 save
pm2 startup
```

`pm2 startup` 会输出一条带 `sudo` 的命令，需要复制并执行该命令，然后再次运行：

```bash
pm2 save
```

日常发布重载：

```bash
pm2 reload deploy/ecosystem.config.cjs --env production
```

## 9. Nginx 配置

项目已准备两份配置：

```text
deploy/nginx/gamified-bootcamp.http.conf
deploy/nginx/gamified-bootcamp.conf
```

第一份用于首次申请证书，第二份是正式 HTTPS 配置。

使用前必须把模板中的：

```text
camp.example.com
```

替换为真实已备案域名。

首次部署先复制 HTTP 引导配置：

```bash
sudo mkdir -p /var/www/certbot
sudo cp deploy/nginx/gamified-bootcamp.http.conf /etc/nginx/sites-available/gamified-bootcamp
sudo ln -s /etc/nginx/sites-available/gamified-bootcamp /etc/nginx/sites-enabled/gamified-bootcamp
sudo nginx -t
sudo systemctl reload nginx
```

模板的代理目标固定为：

```text
http://127.0.0.1:3000
```

证书签发完成后换成最终 HTTPS 配置：

```bash
sudo cp deploy/nginx/gamified-bootcamp.conf /etc/nginx/sites-available/gamified-bootcamp
sudo nginx -t
sudo systemctl reload nginx
```

如果 Ubuntu 默认站点占用端口，可移除默认软链接：

```bash
sudo rm /etc/nginx/sites-enabled/default
```

## 10. 防火墙和安全组

云服务器安全组：

| 端口 | 来源 | 用途 |
| --- | --- | --- |
| 22/TCP | 仅管理员固定 IP | SSH |
| 80/TCP | `0.0.0.0/0`、`::/0` | HTTP 和证书验证 |
| 443/TCP | `0.0.0.0/0`、`::/0` | HTTPS |

禁止公网开放：

- `3000`：仅由本机 Nginx 访问。
- `5432`：未来 PostgreSQL 只允许应用服务器内网访问。
- PM2 管理端口：不对公网开放。

Ubuntu UFW 示例：

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
sudo ufw status
```

## 11. 域名解析步骤

1. 在云平台获取服务器公网 IPv4。
2. 进入域名 DNS 控制台。
3. 新增 A 记录：

```text
主机记录：camp
记录类型：A
记录值：<SERVER_PUBLIC_IP>
TTL：600
```

4. 最终访问域名为：

```text
camp.example.com
```

5. 如需根域名访问，再新增：

```text
主机记录：@
记录类型：A
记录值：<SERVER_PUBLIC_IP>
```

6. 等待 DNS 生效并检查：

```bash
nslookup camp.example.com
```

DNS 解析结果必须与服务器公网 IP 一致。

## 12. HTTPS 证书配置

建议使用 Certbot 自动申请和续期证书。

安装：

```bash
sudo apt install -y certbot python3-certbot-nginx
```

首次签发证书前，先启用 `deploy/nginx/gamified-bootcamp.http.conf`，确保域名已经解析到服务器。

申请：

```bash
sudo mkdir -p /var/www/certbot
sudo certbot certonly --webroot -w /var/www/certbot -d camp.example.com
```

证书生成后路径为：

```text
/etc/letsencrypt/live/camp.example.com/fullchain.pem
/etc/letsencrypt/live/camp.example.com/privkey.pem
```

确认路径与 `deploy/nginx/gamified-bootcamp.conf` 一致，再切换最终配置：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

检查自动续期：

```bash
sudo certbot renew --dry-run
systemctl status certbot.timer
```

也可使用阿里云或腾讯云 SSL 证书服务。使用云厂商证书时，需要将证书文件放到仅 root 可读的目录，并同步修改 Nginx 证书路径。

## 13. 后续 PostgreSQL 接入

当前版本不需要数据库。未来接入真实数据时建议：

- 阿里云使用 RDS PostgreSQL。
- 腾讯云使用 TencentDB for PostgreSQL。
- PostgreSQL 与应用服务器同地域、同 VPC。
- 不向公网开放 `5432`。
- 使用独立应用账号和最小权限。
- 开启自动备份、时间点恢复和慢查询日志。
- 通过服务端环境变量注入连接串。

示例仅使用占位符：

```env
DATABASE_URL=postgresql://app_user:<REDACTED>@postgres.internal:5432/gamified_bootcamp
```

接入 PostgreSQL 属于后续开发，不在本次部署配置范围内。

## 14. 日常发布步骤

```bash
cd /opt/gamified-bootcamp/current
git pull --ff-only
pnpm install --frozen-lockfile
pnpm test
pnpm build
pm2 reload deploy/ecosystem.config.cjs --env production
pm2 status
curl -I http://127.0.0.1:3000/app/home
```

不要在 `pnpm dev` 正在使用同一个项目目录时执行 `pnpm build`。

## 15. 回滚方案

### 发布前

记录当前稳定 commit：

```bash
git rev-parse HEAD
```

建议为每个生产版本创建 tag：

```bash
git tag production-YYYYMMDD-HHMM
git push origin production-YYYYMMDD-HHMM
```

### 回滚

```bash
cd /opt/gamified-bootcamp/current
git fetch --tags
git checkout <LAST_GOOD_COMMIT_OR_TAG>
pnpm install --frozen-lockfile
pnpm test
pnpm build
pm2 reload deploy/ecosystem.config.cjs --env production
```

验证：

```bash
pm2 status
curl -I http://127.0.0.1:3000/app/home
```

未来涉及数据库迁移时，必须先备份数据库；禁止在没有备份和恢复方案时执行破坏性回滚。

## 16. 部署检查清单

### 服务器

- [ ] 使用阿里云或腾讯云中国大陆地域服务器。
- [ ] 至少 2 核 4 GB、60 GB SSD、5 Mbps 带宽。
- [ ] Ubuntu、Node.js 22、pnpm、PM2、Nginx 已安装。
- [ ] 系统补丁已更新。

### 构建与进程

- [ ] `pnpm install --frozen-lockfile` 通过。
- [ ] `pnpm test` 通过。
- [ ] `pnpm build` 通过。
- [ ] `pm2 status` 显示 `online`。
- [ ] `pm2 save` 和开机启动已配置。
- [ ] `curl http://127.0.0.1:3000/app/home` 返回成功。

### 网络与安全

- [ ] 安全组只开放必要的 `22/80/443`。
- [ ] `22` 端口限制管理员来源 IP。
- [ ] `3000/5432` 未向公网开放。
- [ ] Nginx 反向代理到 `127.0.0.1:3000`。
- [ ] 防火墙已启用。

### 域名与 HTTPS

- [ ] 域名已完成 ICP 备案。
- [ ] DNS A 记录指向服务器公网 IP。
- [ ] HTTPS 证书有效。
- [ ] HTTP 自动跳转 HTTPS。
- [ ] 证书自动续期测试通过。

### 页面

- [ ] `/` 可以打开并跳转到首页。
- [ ] `/app/home` 正常。
- [ ] `/app/course` 正常。
- [ ] `/app/course/day-4` 正常。
- [ ] `/app/submit/day-4` 正常。
- [ ] `/app/growth` 正常。
- [ ] 页面刷新无 404。
- [ ] 手机网络下访问稳定。
- [ ] 当前版本没有请求 Supabase。

### 运维

- [ ] PM2 日志目录可写。
- [ ] 磁盘和内存监控已开启。
- [ ] Nginx 无持续 `502/504`。
- [ ] 已记录回滚 commit 或 tag。
- [ ] 已完成一次回滚演练。

## 17. 本地开发服务器规则

以后完成开发后：

1. 默认执行 `pnpm dev`。
2. 保持开发服务器持续运行。
3. 返回 Local 和 Network 地址。
4. 不主动停止开发服务器。
5. 只有用户明确说“停止服务”时才关闭。

默认地址：

```text
Local:   http://localhost:3000
Network: http://<LOCAL_LAN_IP>:3000
```
