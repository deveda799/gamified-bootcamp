# AI 开发上下文

## 文档规范

1. 所有 Markdown 文档统一放在 `docs/`。
2. 所有项目状态统一维护在 `docs/02_PROJECT_STATUS.md`。
3. 所有路线图统一维护在 `docs/03_ROADMAP.md`。
4. 每完成一个 Sprint，必须更新 `docs/02_PROJECT_STATUS.md`。
5. 不允许重复创建同类型文档；新增前先检查现有目录和文档。
6. 不允许把临时文档散落到项目根目录、`superpowers` 或其它临时目录。

## 开发原则

- 编码前明确假设、计划和验收条件。
- 只做必要功能，避免过度抽象和无关扩展。
- 精准修改，保留现有代码风格，不改无关文件。
- 小步验证，达到当前目标即停止。
- 不泄露真实密钥、密码或令牌；示例必须使用假值或 `<REDACTED>`。

## 文档分类

- 产品需求和原型：`docs/产品/`
- 开发架构、API、数据库和运行说明：`docs/开发/`
- Sprint 计划与阶段记录：`docs/Sprint/`
- 训练营课程资料：`docs/训练营/`
- 游戏规则：`docs/游戏规则/`
- Bug 记录：`docs/Bug/`
- 部署与运维：`docs/部署/`
