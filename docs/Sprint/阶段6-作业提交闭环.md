# 阶段 6：作业提交闭环

## 已实现

- `POST /api/submissions/draft` 从占位改为服务端保存草稿。
- `POST /api/submissions/submit` 从占位改为服务端正式提交。
- 使用 Supabase Auth 获取当前用户。
- 使用 service role 服务端写入作业和积分，避免前端伪造。
- 草稿保存不会发积分。
- 首次正式提交作业后写入 `point_ledger`。
- 重复提交同一作业不会重复发积分。
- 作业提交积分 event key 为：

```text
assignment_submitted:{enrollmentId}:{assignmentId}
```

- 提交后根据 V1 徽章规则检查新徽章，例如累计提交 3 次作业触发“项目猎人”。
