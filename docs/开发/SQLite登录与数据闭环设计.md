# SQLite 登录与数据闭环设计

## 目标

在腾讯云单机环境中提供不依赖海外服务的 H5 MVP：学员使用唯一昵称登录，签到、作业、积分和排行榜数据保存在服务器本地 SQLite，管理员使用独立密码查看数据。

## 范围

- 新增 `/login` 昵称登录页。
- 昵称全局唯一；首次登录自动创建账号，后续登录原账号。
- 使用 HttpOnly Cookie 保存服务端会话，不把 localStorage 作为唯一数据源。
- 签到每天仅奖励一次积分。
- 作业按课程每天仅提交并奖励一次，保存文字和链接；图片继续保留占位，不实现上传。
- 首页积分、成长页排行榜和后台数据从 SQLite 读取。
- `/admin` 使用环境变量 `ADMIN_PASSWORD` 保护。
- 不接 Supabase、微信、短信、GitHub 或其他海外运行时服务。

## 技术设计

- 使用 Node.js 24 内置 `node:sqlite`，数据库默认保存到 `data/gamified-bootcamp.sqlite`，可通过 `SQLITE_PATH` 覆盖。
- 数据表：
  - `users`：学员 ID、唯一昵称、创建时间。
  - `sessions`：随机会话令牌摘要、学员 ID、到期时间。
  - `check_ins`：学员、签到日期、积分。
  - `submissions`：学员、课程 ID、文字、链接、提交时间、积分。
- 积分由签到和作业记录聚合，唯一约束防止重复奖励。
- Next.js Route Handlers 提供登录、退出、当前学员、签到、作业、排行榜和后台只读接口。
- 学员接口从 HttpOnly Cookie 解析会话；无会话时返回 401，页面跳转 `/login`。
- 管理员使用独立 HttpOnly Cookie；密码只从服务端环境变量读取。

## 错误处理

- 昵称去除首尾空格，长度限制为 2–20 个字符。
- 重复操作返回当前状态，不重复写入积分。
- SQLite 文件或环境变量异常时返回安全错误，不向浏览器暴露文件路径或密码。

## 验收

- `/`、`/app/home`、`/app/course`、`/app/growth` 返回 200。
- 首次昵称登录创建账号，再次使用同名进入同一账号。
- 签到、作业、积分在刷新和更换浏览器后仍由服务器保存。
- 多个学员的数据出现在服务端排行榜和管理员页面。
- 生产环境仅依赖 Node.js、PM2、Nginx 和本地 SQLite。
