# 阶段 10：依赖安装与首次构建验证

## 本阶段目标

完成 MVP V1 项目的本地依赖安装，并执行首次 Next.js 生产构建验证。

## 已完成内容

1. 安装项目依赖
   - 使用 `npm.cmd install --loglevel verbose`
   - 生成 `node_modules`
   - 生成 `package-lock.json`

2. 执行首次生产构建
   - 使用 `npm.cmd run build`
   - 生成 `.next`

3. 修复 Next.js 15 构建暴露的问题
   - 动态页面和动态 API 路由的 `params` 类型改为 Promise，并在函数内 `await params`
   - Supabase SSR server client 的 cookie `setAll` 参数补充显式类型
